什么是HSL?

2021年8月30日
| CSS
| 阅读

我相信您已经熟悉使用RGB和HEX在CSS中表示颜色，但这些并不是在CSS中编写颜色的最佳方法。HEX和RGB非常难以理解，如您尝试使颜色变暗、变亮或者将颜色从红色转换为橙色并不是很容易。而HSL是一种更容易阅读的格式，它使得更改颜色的亮度或者色调等操作非常简单。

什么是HSL?

HSL(hue, saturation, lightness)代表色相，饱和度和亮度。这类似于RGB通过红色、绿色、蓝色的值组合在一起来表示颜色，但使用HSL，这些值更加直观。

色相

颜色的色相由围绕圆圈的度数表示。试想一个色轮，其中0度表示红色，180度表示青色，240度表示蓝色，360度表示红色。

饱和度

饱和度由 0 到 100 之间的百分比表示，并确定颜色的灰色程度。0％ 表示灰色阴影，而 100％ 是全色。

亮度

亮度也是用百分比表示。如果颜色是50%的亮度，那么意味着没有额外的白色或黑色添加到颜色中。当百分比增加到50%以上时，它将会为颜色添加更多的白色，直到100%亮度时会完全白色。当百分比降低到50%以下时，它将会为颜色添加更多黑色，直到0%亮度时将会完全黑色。

如何使用HSL?

我们了解了什么是HSL，现在我们可以在CSS中尝试使用HSL。

.class {
	/* 红色 #FF0000 */
    background-color: hsl(0, 100%, 50%);
}

传递给hsl()的第一个值是色相，它是介于0到360之前的度数值，第二个值是饱和度，它是介于0%到100%之间的百分比值，最后一个值是亮度，它是0%到100%之间的百分比值。您还可以使用hsla()来表示部分透明的颜色，该函数接收第四个参数，它是一个介于0到1之间的值，其中1表示完全不透明，0表示完全透明。

为什么HSL很重要？

像这样书写颜色咋一看似乎有点奇怪，因为您很可能已经习惯了HEX或RGB,，但是一旦您习惯了HSL，就会使编写CSS变得更加有趣。以下是几个原因：

轻松改变亮度/饱和度

正如我在本文开头提到的，您经常需要在CSS中更改颜色的亮度或饱和度，例如实现更改按钮悬停状态背景色之类的操作。使用HEX可能会很烦，因为您压根不知道原本颜色暗10%的颜色值是多少。如果使用HSL，假如有背景色hsl(200, 100%, 50%)，您将很容易计算出暗10%的背景色，您只需将亮度值降低10%即可，得到hsl(200, 100%, 40%)。

您甚至可以更进一步地，在CSS中使用calc()和自定义属性来优化您的代码。例如：

.btn {
  --background-hue: 200;
  --lightness-offset: 0%;
  background-color: hsl(
    var(--background-hue),
    100%,
    calc(50% + var(--lightness-offset))
  );
}

.btn:hover {
  --lightness-offset: -10%;
}

.btn:focus {
  --lightness-offset: -20%;
}

.btn-danger {
  --background-hue: 0;
}

通过少量的代码，我们编写了一个基本的btn样式类，它在悬停时变暗10%，在聚焦时变暗20%。

轻松获取相关颜色

如果您熟悉一点关于色彩的理论，那您可能了解互补色、对比色和类似色等，这些不太类型的颜色用GRB或HEX很难弄清楚，但使用HSL很容易找到诸如互补色之类的颜色，因为互补色只是色轮上另一侧的颜色，在HSL的情况下，这意味着它们的色相之间相差180。

轻松改变色相

最后，HSL重要的一点是可以轻松更改颜色的色相。如果您想让您的红色变得橙色一点，你可以通过让你的色相更大一点轻松完成，因为橙色在色轮上紧随红色之后。

总结

HSL 是一种在 CSS 中表示颜色的不同方式，它不仅更易于阅读，而且在 CSS 中更易于修改和使用。在我看来，它是 CSS 中最好的颜色格式。
